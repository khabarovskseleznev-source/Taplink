# Схема деплоя и хостинга — seleznev.site

## Домен

- **Домен:** `seleznev.site`
- **Регистратор / DNS:** REG.RU
- REG.RU — **только DNS**, файлы сайта там не хранятся и загружать ничего туда не нужно

Эти записи перенаправляют весь трафик с `seleznev.site` на GitHub Pages.

## Хостинг

- **Платформа:** GitHub Pages
- **Репозиторий:** `https://github.com/khabarovskseleznev-source/Taplink`
- **Ветка деплоя:** `main`
- **CNAME файл:** в корне репозитория, содержит `seleznev.site`

## Деплой

Деплой **только один** — через `git push` в ветку `main`.

```bash
git add .
git commit -m "Описание изменений"
git push
```

После пуша GitHub Pages автоматически публикует изменения в течение 1–2 минут. REG.RU при этом трогать не нужно.

## Схема (кратко)

```
Браузер → seleznev.site
             ↓
        REG.RU DNS
        (A-записи → GitHub IPs)
             ↓
        GitHub Pages
        (репозиторий: khabarovskseleznev-source/Taplink, ветка: main)
             ↓
        Сайт отдаётся пользователю
```

## Важно

- Никогда не трогать DNS-записи на REG.RU без необходимости
- CNAME файл в репозитории обязателен — без него GitHub Pages не подключит кастомный домен
- Если домен перестал работать — сначала проверить CNAME файл в репозитории, потом DNS на REG.RU
